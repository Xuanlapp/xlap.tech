## 文檔維護

- 所有技術文檔都應該放在此 `docs/` 目錄中
- 項目主要的 README.md 保留在根目錄
- 請保持文檔的及時更新，記錄重要的開發過程和決策

## 文檔格式

建議使用 Markdown 格式撰寫文檔，包含：
- 清楚的標題結構
- 代碼示例使用適當的語法高亮
- 問題描述、原因分析、解決方案的完整記錄
- 必要時加入圖表或流程圖
